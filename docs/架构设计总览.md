# DAI 实验平台 — 架构设计总览

## 一、平台定位

一个面向 **AI/机器学习/深度学习教学** 的在线实验平台。学生在浏览器中即可完成理论学习、代码实验、作业提交与自动判题，无需安装任何本地环境。

**核心目标：** 打通 "理论学习 → 动手实验 → 自动判题 → 成绩反馈" 的完整教学闭环。

---

## 二、目标用户

| 角色 | 核心场景 |
|------|---------|
| **学生** | 加入课程、学习章节内容、在 JupyterLab 中做实验、提交编程作业、查看判题结果和成绩 |
| **教师/助教** | 创建课程、管理班级、布置作业和考试、查看学情数据 |
| **管理员** | 管理用户账号、配置系统参数、管理计算资源、监控平台运行 |
| **实验内容开发者** | 开发新的实验模块（如可视化实验、判题题库），扩展平台能力 |

---

## 三、核心功能模块

```
DAI 实验平台
├── 用户认证与权限管理
│   ├── 注册/登录/登出
│   ├── JWT Token 认证
│   └── 角色权限控制（学生/教师/管理员/开发者）
│
├── 课程管理
│   ├── 教师创建课程、编辑章节与课时
│   ├── 学生选课/退课
│   └── 课时内容支持视频、Markdown 文本、Notebook 等多种形式
│
├── 作业与在线判题
│   ├── 教师布置编程作业（设置题目、测试用例、截止时间）
│   ├── 学生在 Web 端编写 Python 代码并提交
│   ├── Docker 沙箱隔离执行，自动判题（安全、资源受限）
│   └── 即时反馈：正确/错误/超时，展示详细测试结果
│
├── JupyterLab 交互式实验
│   ├── JupyterLab 以 iframe 嵌入平台前端
│   ├── 学生无需离开平台即可使用 Notebook 进行数据分析、模型训练
│   └── 预置 AI/ML 教学 Notebook 模板
│
├── 考试系统
│   ├── 限时考试、自动收卷
│   └── 成绩统计与导出
│
└── 实验模块（可扩展）
    └── 预留接口，可接入 Swin Transformer 可视化等专项实验模块
```

## 四、技术架构

### 整体拓扑

```

│                   浏览器 (学生/教师)               │
│   ┌──────────────────────────────────────────┐   │
│   │          Vue.js 前端（另一位同学负责）      │   │
│   │  用户界面 │ 代码编辑器 │ JupyterLab iframe │   │
│   └──────────────┬───────────────────────────┘   │
└──────────────────┼──────────────────────────────┘
                   │ HTTP + JWT Token
┌──────────────────▼──────────────────────────────┐
│              Nginx 反向代理                       │
│   /api/* → 后端     /jupyter/* → JupyterLab      │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│              FastAPI 后端（Python）               │
│                                                  │
│   ┌──────────┐  ┌──────────┐  ┌──────────────┐  │
│   │ 用户模块  │  │ 课程模块  │  │ 判题服务     │  │
│   │ 认证/权限 │  │ 作业/考试 │  │ Docker 调度  │  │
│   └─────┬────┘  └─────┬────┘  └──────┬───────┘  │
│         │             │              │           │
│         └─────────────┼──────────────┘           │
│                       │                          │
│   ┌───────────────────▼───────────────────────┐  │
│   │        Redis 队列 Worker（独立进程）         │  │
│   │     消费判题任务 / 管理 Docker 容器生命周期  │  │
│   └───────────────────────────────────────────┘  │
└──────┬──────────────┬──────────────┬─────────────┘
       │              │              │
┌──────▼──────┐ ┌─────▼─────┐ ┌─────▼──────────┐
│   MySQL     │ │   Redis   │ │  Docker 沙箱    │
│  业务数据    │ │  缓存/队列 │ │  仅 Python 镜像  │
│             │ │  Session  │ │  内存限制 256MB  │
│  用户/课程   │ │  判题结果  │ │  CPU 限制 1核   │
│  作业/成绩   │ │           │ │  无网络/只读    │
└─────────────┘ └───────────┘ └────────────────┘
```

### 技术选型

| 层面 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue.js | 另一位同学负责 |
| 后端 | FastAPI（Python） | 自动生成 Swagger 接口文档，前端可直接参考对接 |
| 业务数据库 | MySQL | 用户、课程、作业、成绩等持久化数据 |
| 缓存/队列 | Redis | 用户会话、JWT 黑名单、判题任务队列 |
| 认证 | JWT + Redis | 无状态 Token 认证，Redis 维护黑名单支持主动登出 |
| 判题隔离 | Docker | 每次提交启动临时容器执行，执行完自动销毁 |
| 交互实验 | JupyterLab | 嵌入前端，学生在线编写和运行 Python/ML 代码 |
| 版本管理 | Git | 代码版本控制 |
| 协作文档 | 腾讯文档 | 需求分析、系统设计文档维护 |

## 五、判题流程（关键链路）

```
学生提交代码
    │
    ▼
① 后端接收 → 存入 MySQL → 任务推入 Redis 队列 → 返回"判题中"
    │
    ▼
② Worker 进程从 Redis 队列取出任务
    │
    ▼
③ 启动 Docker 容器执行代码
   ├── 内存限制 256MB
   ├── CPU 限制 1 核
   ├── 禁止网络访问
   ├── 文件系统只读
   └── 超时 10 秒强杀
    │
    ▼
④ 收集 stdout / stderr / 退出码 / 执行时间
    │
    ▼
⑤ 比对测试用例预期输出 → 判题结论
   ├── ✅ Accepted（通过）
   ├── ❌ Wrong Answer（答案错误）
   ├── ❌ Time Limit Exceeded（超时）
   └── ❌ Runtime Error（运行错误）
    │
    ▼
⑥ 结果写回 MySQL + Redis 缓存 → 前端展示给学生
```

## 六、数据库核心实体

```
users（用户）
  └── courses（课程）
        ├── chapters（章节）
        │     └── lessons（课时）
        ├── course_enrollments（选课记录）
        ├── assignments（作业）
        │     ├── judge_questions（判题题目 + 测试用例）
        │     └── submissions（学生提交 + 判题结果）
        └── exams（考试）
              ├── exam_questions（考试题目）
              ├── exam_submissions（考试提交）
              └── exam_grades（考试成绩）

experiment_modules（实验模块，如 Swin Transformer 可视化）
  └── experiment_records（学生实验记录）
```

## 七、开发计划

| 阶段 | 内容 | 说明 |
|------|------|------|
| **阶段 1** | 项目骨架 + 用户认证 | FastAPI 项目初始化、JWT 登录注册、角色权限 |
| **阶段 2** | Docker 判题沙箱 | 判题镜像构建、Worker 进程、代码安全执行与判题 |
| **阶段 3** | JupyterLab 集成 | Jupyter 服务部署、反向代理、Notebook 模板 |
| **阶段 4** | 课程与作业 CRUD | 课程管理、作业布置与提交、成绩查询 |
| **阶段 5** | 联调与文档 | Swagger 完善、前后端接口对接 |
